# < 스크립트 언어 >

## 01. 스크립트 언어(Script Language)

* 스크립트 언어는 HTML 문서 안에 직접 프로그래밍 언어를 삽입해 사용하는 언어.
  * 기계어로 컴파일되지 않고, 별도의 번역기가 소스를 분석해 작동함.
  * 게시판 입력, 상품 검색, 회원 가입 등과 같은 DB 처리 작업을 수행하기 위해 주로 사용.
* **서버용 스크립트 언어**와 **클라이언트용 스크립트 언어**로 분류됨.
  * 서버용 스크립트 언어
    * 서버에서 해석되어 실행된 후 결과만 클라이언트로 보냄.
    * ASP, JSP, PHP, Python이 해당됨.
  * 클라이언트용 스크립트 언어
    * 클라이언트의 웹 브라우저에서 해석 및 실행됨.
    * JAVA Script, Visual Basic Script가 해당됨.

---

## 02. 스크립트 언어의 종류

### 02-1. 자바 스크립트(JAVA Script)

* **웹 페이지의 동작 제어**에 사용되는 클라이언트용 스크립트 언어.
* 클래스가 존재하지 않고, 변수 선언도 필요 없음.
* 서버에서 데이터를 전송할 때 마다 아이디, 비밀번호, 수량 등의 입력 사항을 확인하기 위해 많이 사용됨.

### 02-2. 비주얼 베이직 스크립트(Visual Basic Script)

* MS 사에서 자바 스크립트에 대응하기 위해 제작한 언어.
* **Active X**를 사용해 MS 사의 애플리케이션 컨트롤 가능.

### 02-3. ASP(Active Server Page)

* 서버 측에서 동적으로 수행되는 페이지를 만들기 위한 언어.
* MS 사에서 제작함.
* Windows 계열에서만 사용 가능.

### 02-4. JSP(Java Server Page)

* JAVA로 만들어진 서버용 스크립트 언어.
* 다양한 운영체제에서 사용 가능.

### 02-5. PHP(Professional Hypertext Preprocessor)

* Linux, Unix, Windows 운영체제에서 사용 가능한 서버용 스크립트 언어.
* C, Java 등과 문법이 유사해 배우기 쉬워 웹 페이지 제작에 많이 사용됨.

### 02-6. Python

* 객체지향 기능을 지원하는 대화형 인터프리터 언어.
  * 인터프리터 언어: 원시 프로그램을 줄 단위로 번역해 바로 실행하는 언어.
  * 목적 프로그램을 생성하지 않고 즉시 실행 결과를 출력.
* 플랫폼에 독립적이고, 문법이 간단해 배우기 쉬움.

### 02-7. 쉘 스크립트

* Unix/Linux 계열의 shell에서 사용되는 명령어들의 조합으로 구성된 스크립트 언어.
* 선택형 제어문: if, case
* 반복형 제어문: for, while, until

### 02-8. Basic

* 절차지향 기능을 지원하는 대화형 인터프리터 언어.
* 초보자도 쉽게 사용할 수 있는 문법 구조.

## 03. XML, JSON, AJAX

### 03-1. XML(eXtensible Markup Language)

* 특수한 목적을 갖는 마크업 언어를 만드는 데 사용되는 다목적 마크업 언어.
* 웹 페이지의 기본 형식인 HTML의 문법이 각 웹 브라우저에서 상호 호환적이지 못한 문제와, SGML의 복잡함을 해결하기 위해 개발됨.

### 03-2. JSON(JavaScript Object Notation)

* 속성-값 쌍(Attribue-Value Pairs)로 이루어진 데이터 객체를 전달하기 위해, 사람이 읽을 수 있는 텍스트를 사용한 개방형 표준 포맷.
* 비동기 처리에 사용되는 AJAX에서 XML을 대체해 사용하고 있음.

### 03-3. AJAX(Asynchronous JavaScript and XML)

* 자바 스크립트 등을 이용해 클라이언트와 서버 간 XML 데이터 교환 및 제어함으로써, 이용자가 웹 페이지와 자유롭게 상호 작용할 수 있는 비동기 통신 기술.
